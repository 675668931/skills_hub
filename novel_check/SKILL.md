# 项目级检查器

对已生成的小说章节进行宏观检查，发现单章检查无法发现的累积性问题。支持增量检查和跨报告问题追踪。

## 快速参考

| 场景 | 操作 |
|------|------|
| 用户要求检查 | 扫描目录查找已有报告，继续检查 |

---

## 工作流

```
用户触发检查
    ↓
扫描 output/ 查找已有报告
    ↓
确定检查起点和范围
    ↓
读取项目文件
    ↓
执行六项检查
    ↓
生成检查报告
```

---

## 第一步：扫描已有报告

扫描 `output/` 目录，查找已有的检查报告。

```
=== 扫描已有报告 ===

【查找结果】
output/ 目录下共有 X 个检查报告：
- 检查报告_第01章_第20章.md
- 检查报告_第21章_第40章.md
- ...

【最新报告】
- 文件名：检查报告_第21章_第40章.md
- 检查范围：第21章 ~ 第40章
- 上次发现问题：X个（未修复：X个）

【确定本次检查范围】
- 最新章节：第XX章
- 已检查至：第40章
- 本次应检查：第41章 ~ 第XX章（共XX章）
```

::: warning
**【强制】必须扫描 output/ 目录，如无报告则从第1章开始检查。**
:::

---

## 第二步：读取项目文件

读取以下文件进行综合分析：

```
output/大纲.md              → 主线进度对照
output/提示词.md            → 设定基准
output/第XX章_*.md          → 需要检查的章节
output/检查报告_*.md         → 已有报告（用于问题追踪）
.learnings/CHARACTERS.md    → 角色状态记录
.learnings/PLOT_POINTS.md   → 情节事件记录
.learnings/STORY_BIBLE.md   → 世界观设定
.learnings/ERRORS.md        → 已记录的失败/问题
```

::: warning
**【强制】必须完整读取以上所有文件，未读取禁止开始检查。**
:::

---

## 第三步：确定检查范围

```
【本次检查范围】
- 起始章节：第XX章
- 截止章节：第XX章
- 新增章节数：XX章
- 检查模式：[basic / medium / full]
  - basic：主线推进 + 逻辑一致性 + 角色戏份
  - medium：+ 伏笔回收 + 爽点节奏
  - full：+ 世界观自洽 + 势力关系 + 详细报告
```

---

## 第四步：六项检查

### 检查一：主线推进检查

```
=== 主线推进检查 ===

【本次新增章节】
- 第XX章：完成事件 [描述]
- 第XX章：完成事件 [描述]
- ...

【大纲对照】
- 本次应完成：[根据大纲]
- 实际完成：[列举]

【偏离判定】
□ 无偏离
□ 轻度偏离
□ 严重偏离

【如有偏离】
- 偏移事件：
- 大纲预期：
- 修正建议：
```

### 检查二：逻辑一致性检查

```
=== 逻辑一致性检查 ===

【矛盾列表】
| 章节 | 矛盾点A | 矛盾点B | 严重程度 |
|------|---------|---------|----------|
| 第X章 | [设定A] | [设定B] | high |

【历史矛盾追踪】（如有上期报告）
| 矛盾ID | 矛盾描述 | 上次状态 | 本次状态 |
|--------|---------|----------|----------|
| LOGIC-001 | [描述] | 未修复 | 已修复/仍存在 |

【判定】
□ 无矛盾
□ 有矛盾（见上表）
```

**常见矛盾类型：**

| 类型 | 表现 | 严重程度 |
|------|------|----------|
| 境界矛盾 | 主角境界忽高忽低 | critical |
| 时间线矛盾 | 事件发生顺序前后不一 | high |
| 人物关系矛盾 | 前后关系描述不一致 | high |
| 地理矛盾 | 地点相对位置变化 | medium |
| 实力矛盾 | 某人实力前后描述不符 | high |

### 检查三：角色戏份检查

```
=== 角色戏份检查 ===

【主角戏份占比】
- 主角出场章节：X/Y章
- 主角主导情节章节：X/Y章
- 占比：XX%（目标>60%）

【配角戏份TOP5】
| 角色 | 出场章节数 | 互动次数 | 戏份占比 |
|------|-----------|----------|----------|
| 角色A | X | X | XX% |

【问题检测】
□ 配角抢戏（配角主导情节超过3章）
□ 角色蒸发（大纲要求出现但未出场）
□ 戏份失衡（主角戏份<50%）

【判定】
□ 正常
□ 需要修正
```

### 检查四：伏笔回收检查

```
=== 伏笔回收检查 ===

【伏笔统计】
- 总埋设伏笔：X条
- 已回收：X条（X%）
- 待回收：X条
- 失效伏笔：X条

【伏笔状态表】
| 伏笔ID | 埋设章节 | 内容摘要 | 状态 |
|--------|---------|---------|------|
| PLOT-XXX | 第X章 | [内容] | 已回收/待回收/失效 |

【判定】
□ 回收率正常（>70%）
□ 回收率偏低（50-70%）
□ 回收率过低（<50%）
```

### 检查五：爽点节奏分析

```
=== 爽点节奏分析 ===

【爽点统计】
- 总爽点：X个
- 打脸类：X个（XX%）
- 突破类：X个（XX%）
- 获宝类：X个（XX%）
- 逆转类：X个（XX%）

【节奏密度】
- 连续无爽点章节：X章（上限2章）
- 密集爽点章节：X章

【问题检测】
□ 套路重复（同一套路>30%）
□ 节奏疲劳（连续无爽点>2章）
□ 类型失衡（某类型>50%）

【判定】
□ 节奏健康
□ 存在风险
```

### 检查六：世界观自洽检查

```
=== 世界观自洽检查 ===

【核心规则检查】
| 规则 | 状态 | 违反章节 |
|------|------|----------|
| 规则1 | ✓ | - |
| 规则2 | ✗ | 第X章 |

【势力关系】
- [势力A] —[对立]→ [势力B]
- [势力A] —[盟友]→ [势力C]

【判定】
□ 自洽
□ 存在问题
```

### 检查七：作者/叙事者视角跳脱检查

```
=== 作者/叙事者视角跳脱检查 ===

【检测目标】
AI用了作者/叙事者视角，而不是角色视角。
旁白中用"主角"或第三人称描述本应用第一人称或角色名描述的内容，说明AI在以旁观者身份讲故事。

【检测方法】
让AI直接判断：这段文字是"角色在讲自己的故事"还是"旁白在讲角色的故事"？
旁白中用"主角"来指代角色，即为跳脱。

【检测结果】
| 章节 | 跳脱次数 | 典型位置 |
|------|----------|----------|
| 第X章 | X处 | 第X行、第X行... |

【判定】
□ 合格
□ 不合格（发现X处）
